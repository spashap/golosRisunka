# Голос рисунка — Бренд-бук (дизайн-система)

> Источник истины для ВСЕГО визуала сайта. Утверждён заказчиком 18.06.2026.
> Концепция выросла из hero-фото (`static/img/hero.jpg`, «тёплый золотой час»):
> родитель и ребёнок в тёплом свете + детские рисунки.
>
> **ГЛАВНЫЙ ЗАКОН.** Один глобальный CSS-источник: `static/css/tokens.css` (значения) +
> `static/css/components.css` (компоненты). Ни одна страница не задаёт цвета/шрифты/размеры
> сама — только через токены и существующие компоненты. Любой новый элемент = композиция
> компонентов; не хватает — добавляешь компонент в `components.css`, не инлайнишь в страницу.
> Меняешь токен → меняется ВЕСЬ сайт. Это и есть требование «нет отдельных контролов».

---

## 1. Голос бренда
Тёплый, заботливый, спокойный. «Тёплый отчёт, который вам спокойно протягивает понимающий
человек». Не клиника, не игрушка, не «нейросеть». Спокойствие важнее украшений: один акцент в
кадре, воздух, мягкие тени. По обещанию продукта — «по видимой детали, без диагнозов» — дизайн
должен ощущаться ясным и честным, без «магии».

## 2. Цвет (токены в `tokens.css`)

Палитра «Золотой час» (дефолт). Тема переключается классом на `<html>` (`settings.PALETTE`):
`""` дефолт, `cl` светлая, `dk` тёмная. Дизайнить только в токенах — темы наследуют.

| Токен | HEX (дефолт) | Роль |
|---|---|---|
| `--bg` | `#FCEFDF` | фон страницы — тёплая бумага (= фон логотипа, чтобы шапка сливалась с лого) |
| `--bg-soft` | `#FFF8EA` | мягкая секция-«передышка» (`.section--soft`) |
| `--alt` | `#F4EBD8` | фон чередующихся секций (`.section--alt`) |
| `--paper` | `#FFFCF4` | карточки, формы, полароиды |
| `--line` | `#EADDC4` | границы, разделители, рамки карточек |
| `--ink` | `#3A2A1C` | основной текст — **тёплый эспрессо, НЕ холодный синий** |
| `--muted` | `#6E5942` | вторичный текст |
| `--soft` | `#A38B6E` | третичный текст / подписи |
| `--accent` | `#3E4E78` | **ДЖИНСОВЫЙ СИНИЙ — действие**: CTA, ссылки, фокус, активные, заливка данных |
| `--accent-2` | `#5E78B4` | светлее джинсового — ховеры, тинты |
| `--accent2` | `#B9722A` | **МЕДОВЫЙ ЯНТАРЬ — бренд-голос**: слово «голос», лого-акцент, тёплые метки (НЕ крупные действия) |
| `--amber-soft` | `#E7A93C` | светлый янтарь — иконки, мелкий декор |
| `--sage` | `#7E9B63` | шалфей — позитивные «галочки» |
| `--danger` | `#C0392B` | ошибки форм (сдержанно) |
| `--ok` | `#2E7D32` | статус «готово»/успех |

**Правило акцентов:** синий = действие, янтарь = голос/тепло. Не путать: крупные кнопки —
ВСЕГДА джинсовый (`--accent`), янтарь — только акцент-голос и мелкие тёплые метки.

## 3. Типографика

Шрифты self-hosted (`fonts.css`), сабсеты cyrillic+latin (₽ U+20BD обязателен).

| Семейство | Токен | Доступные начертания | Где |
|---|---|---|---|
| Rubik | `--font-head` | **800, 900** | заголовки, кнопки, метки, числа |
| Inter | `--font-body` | 400, 500, 600 | весь текст |
| Caveat | `--font-hand` | 600, 700 | **ТОЛЬКО** слово «голос» (`.hand`) и логотип |

> ⚠️ Rubik доступен только в 800/900 — НЕ использовать `font-weight:700` для Rubik (синтетический
> жирный/фолбэк). Нужен новый вес → пересобрать сабсет (`scripts/build_fonts.py`) и обновить `fonts.css`.

**Дисциплина Caveat:** рукописный — это «голос» бренда, он редкий. Появляется 1–2 раза на экран:
H1-акцент «голос» и лого. Везде остальное (подписи, метки, цитаты, номера, имена) — Rubik/Inter.

### Шкала шрифта — 7 ступеней (использовать ТОЛЬКО их)
| Токен | Размер | Где |
|---|---|---|
| `--fs-display` | `clamp(34px,5vw,56px)` | H1 героя |
| `--fs-h2` | `clamp(27px,3.3vw,40px)` | заголовки секций |
| `--fs-h3` | `21px` | подзаголовки карточек/шагов |
| `--fs-lead` | `20px` | лид, `.sub` |
| `--fs-body` | `17px` | основной текст |
| `--fs-small` | `15px` | подписи, второстепенное |
| `--fs-micro` | `13px` | чипы, бейджи, метки |

Любой размер вне шкалы — ошибка; округляй к ближайшей ступени.

## 4. Геометрия, тени, ритм
- Радиусы: `--radius-card` 20px (карточки), `--radius-sm` 10px (мелочь), `--radius-pill` 99px (кнопки/чипы).
- Тени тёплые и мягкие: `--shadow-card`, `--shadow-pol`, `--shadow-hard` (это «приподнятая» кнопка,
  НЕ жёсткая чёрная — старые `5px 5px 0` убраны). Жёстких чёрных теней в бренде нет.
- Контейнер: `.wrap` max-width `--wrap-max` (1160px), боковой паддинг 24px.
- Ритм секций: `.section` 60px вертикали (мобайл 40px). Связка `h2 → .sub → контент` единая.
- Чередование фона — сдержанное: `--alt`/`--bg-soft` точечно выделяют 1–2 ключевые секции,
  НЕ жёсткое «через одну».
- Один фирменный декор-приём — **полароид/матовая карточка**. Бейджи плоские (без наклона).

## 5. Ключевые компоненты (в `components.css`)
- **Hero** (`.hero` + `.hero__in` + `.hero__card`): кино-фото `hero.jpg` на всю ширину (ВНЕ `.wrap`),
  текст в матовой карточке (`backdrop-filter: blur`) слева. Cold-traffic: герой = хук, в карточке
  только H1 + лид + CTA; детали (PDF/чипы) — в плавающем попапе «что входит» (`.cta-zone`/`.cta-pop`,
  тултип, НЕ меняет размер карточки). Мобайл: карточка прозрачнее (0.58), фото дышит.
- **Кнопки**: `.btn` (джинсовая пилюля, мягкий подъём на hover), `.btn--primary` (крупнее + мягкая
  тень), `.btn--ghost` (пунктир). Все действия — `--accent`.
- **Карточки**: `.card`/`.card--pad` (paper + `--line` + `--shadow-card`).
- **Чипы доверия**: `.chips`/`.chip` (пилюля с ✓).
- **Оценки**: `.score-bar` (трек — тинт `--ink`, заливка — `--accent`, число — Rubik `--accent`).
- **Шаги**: `.steps`/`.step`/`.step__n` (число/✓ в джинсовом круге; `.step__n--ok` шалфейный).
- **Поток**: `.flow`/`.flow-card`/`.flow-card__label` (метка — Rubik caps янтарь).
- **Без мифов**: `.compare`/`.compare-col--no|--yes` (без красного: ✕ серый, ✓ шалфей).
- **Карусели**: `.carousel`/`.carousel-wrap` + стрелки `.car-btn` + точки `.car-dots`/`.car-dot`
  (точки строит JS лендинга; индикатор «листается», важно на мобайле).
- **Отзывы**: `.quote-rotator` (одна цитата, автосмена, точки).
- **Цены**: `.price-grid`/`.price-card` (цена — Rubik `--accent`).
- **FAQ**: `.faq` (details/summary, +/–).
- **Блог**: `.blog-card` (сетка + карусель на лендинге).
- **Футер**: `.foot` (тёплый тёмный `#2A1F15`, на всю ширину; бренд — Caveat).
- **Формы/кабинет/админка**: `.field`/`.input`/`.form-card`/`.order-card`/`.status-pill`/
  `.admin-*` — все на токенах, перекрашиваются автоматически.

## 6. Движение
`.reveal` (+`--d1/--d2/--d3` задержки): мягкое появление при скролле (opacity+translateY),
включается `IntersectionObserver`-ом лендинга. **Уважает `prefers-reduced-motion`** (тогда статично).
Анимируем только `opacity`/`transform` (дёшево для LCP/CPU). Без библиотек.

## 7. Картинки и скорость
- Hero оптимизирован: `hero.jpg` (~186 КБ), `hero.webp` (~121 КБ), мобильные `hero-800.*`.
  В CSS — `image-set(webp, jpg)` по **абсолютному** `/static/...` пути (CSS лендинга инлайнится,
  относительный путь сломался бы). Hero предзагружается (`<link rel=preload as=image fetchpriority=high>`).
- Контентные картинки — `loading="lazy"`. Шрифты — preload только критичных (rubik-900/inter-400/caveat-700).

## 8. Что НЕ трогаем этим бренд-буком
- **PDF-отчёт** (`templates/report.html` + `static/css/report.css`) — отдельный продукт, своя
  вёрстка под WeasyPrint и правила (V17). Этот бренд-бук — про САЙТ, не про PDF.
- Аналитику (`data-ym-goal`), версионирование, шрифтовые сабсеты, механизм тем.

## 9. Governance (как вносить изменения)
1. Меняешь только `tokens.css` (значение) или `components.css` (компонент). Страницы — не трогаешь под цвет/шрифт.
2. Новый блок = композиция существующих компонентов. Нет подходящего — добавь компонент в `components.css`.
3. Размеры — только из 7-ступенчатой шкалы. Цвета — только токены. Caveat — только «голос»+лого. Rubik — только 800/900.
4. После правок шрифтов/CSS отчёта — проверка embedded-fonts PDF (это про report.css, не про сайт).
5. Перед `git push` — `bump_version.py` (минор) и `VERSION` в тот же коммит.
